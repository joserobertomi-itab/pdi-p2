Compressão Delta Modulation (DM)

A DM é a forma mais simples de codificação preditiva (um DPCM de 1 bit): em vez de transmitir cada pixel, transmite-se 1 bit por pixel dizendo se ele é maior ou menor que a predição. Aqui, com passo δ = 2: bit 0 = +2, bit 1 = −2. Taxa de compressão nominal: 8 bits → 1 bit = 8:1 (com perdas).

Convenções adotadas (disciplina PDI)

  1. Varredura raster: linha a linha, da esquerda para a direita.
  2. Seed único: apenas o pixel (1,1) é transmitido intacto.
  3. Predição: o valor reconstruído do pixel à esquerda; na 1ª coluna, o reconstruído do pixel de cima (mesma coluna).
  4. Passo: se x > predição → +2; se x < predição → −2.
  5. Empate (x = predição): olha-se o próximo valor da varredura e adota-se o passo que deixa a reconstrução mais perto dele. Se as duas opções ficarem à mesma distância, adota-se o passo mais favorável ao pixel atual.
  6. Saturação nos dois extremos: resultado fora de [0, 4] trava na borda (negativo vira 0; acima do máximo vira 4).
Ponto que mais cai em prova: a predição usa sempre o valor reconstruído, nunca o original — codificador e decodificador precisam enxergar a mesma coisa. Por isso um erro numa posição se propaga (efeito cascata).

Passo a passo

Linha 1 — seed: reconstruído(1,1) = 0 (intacto). Depois: 3>0→2; 0<2→0; 3>0→2; 0<2→0  ⇒  [0, 2, 0, 2, 0]

Linha 2 — 1ª coluna prediz de cima: 2>0→2. Depois: 2=2 empate, próximo valor é 0 → desce → 0; 0=0 empate, próximo 0 → desce (satura) → 0; 0=0 empate, próximo 3 → sobe → 2; 3>2→4  ⇒  [2, 0, 0, 2, 4]

Linha 3 — 0<2→0. Empates com próximo 0 → descem, saturando em 0: 0; 0; 0; 1>0→2  ⇒  [0, 0, 0, 0, 2]

Por que a linha 3 confunde (comparando com a linha 2, que parece igual): as duas chegam à coluna 4 com empate x = predição = 0, mas o próximo valor difere. Na linha 2 o próximo é 3 (subir para 2 aproxima → sobe), e o último pixel original é 3 (chega a 4) → …2, 4. Na linha 3 o próximo é 1, equidistante de 2 e 0 (|2−1| = |0−1| = 1) → o desempate favorece o pixel atual e mantém 0; e o último original é apenas 1, então o passo de ±2 só alcança 2 → …0, 2. Chegar a [0, 0, 0, 2, 4] seria impossível: o original ali é 1, não 3, e a DM só anda ±2 por pixel.

Linha 4 — 3>0→2; 1<2→0; 1>0→2; 4>2→4; 2<4→2  ⇒  [2, 0, 2, 4, 2]

Linha 5 — 3>2→4; 3<4→2; 4>2→4; 4=4 empate, próximo 4 → sobe e satura no máximo → 4; 4=4 empate, último pixel → mantém o mais favorável → 4  ⇒  [4, 2, 4, 4, 4]

MSE

MSE = Σ(original − reconstruído)² / N²

Erros² por linha: 2 + 9 + 1 + 3 + 2 = 17

MSE = 17 / 25 = 0,68

Os dois artefatos clássicos (para citar na prova)

  • Slope overload (δ pequeno demais): em bordas, a DM só sobe/desce 2 por passo e não acompanha o salto — ex.: linha 4, o pixel 4 reconstruído como 4 só porque a subida começou antes; na linha 5, o 3 após a coluna de cima escura vira 4/2 oscilando.
  • Ruído granular (δ grande demais): em regiões constantes a reconstrução oscila ±δ em volta do valor real — aqui a saturação em 0 e a regra do empate com lookahead reduzem esse efeito (linha 3 gruda no 0).
